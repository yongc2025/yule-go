> **版本**：v1.0

> **面向对象**：外部合作伙伴、投资人、新团队成员

## 一、产品愿景

**一句话定位**：有温度、有信任的本地垂钓出行平台——每个门店是一个独立的"超级门店"。

渔乐出行致力于解决线下渔具门店经营单一、同质化严重、客户粘性低的痛点，依托现有渔具实体店，打造"零售 + 短途旅行"双盈利模式。连接钓场（供给）与钓鱼爱好者（需求），由商家（渔具店等）组织出行服务的预约平台。

**核心理念**：
- **门店是信任单元**，不是匿名商家。用户认识老板、去过店、有归属感
- **平台是发现通道**，帮新用户找到附近好店，帮老用户发现新店
- **私域为主，平台为辅**——80% 流量来自门店老客复购，20% 来自平台发现

## 二、商业模式

### 2.1 平台 + 超级门店混合模式

```
用户打开小程序
    │
    ├── 有主场店？ ──→ 直接进入门店首页（私域体验）
    │                      ├── 老板信息 + 信任背书
    │                      ├── 本周活动（本店线路+团期）
    │                      ├── 我的订单（本店）
    │                      └── 切换门店入口
    │
    └── 没有/新用户 ──→ 平台发现页（附近门店、热门活动）
                           └── 进入某门店 → 自动记录"去过的店"
```

### 2.2 收入模型

| 收入来源 | 说明 |
|:---|:---|
| 出行服务费 | 线路定价（98~238元/人），核心收入 |
| 装备租赁 | 鱼竿/帐篷/天幕/桌椅租赁（25~50元/件） |
| 会员充值 | 三档充值（200/500/1000元），锁定长期消费 |
| 渔具零售 | 门店原有业务，会员享折扣（8.5~9.5折） |

## 三、目标用户画像

### 3.1 三类终端客群

| 客群 | 年龄 | 特征 | 核心需求 | 决策特点 |
|:-----|:-----|:-----|:---------|:---------|
| 🎣 垂钓爱好者 | 25-60 | 男性为主，周末时间充裕 | 省心垂钓体验，找到好钓场 | 对鱼情敏感，复购率高，口碑传播主力 |
| 👨‍👩‍👧 亲子家庭 | 28-45 | 携带 5-14 岁儿童 | 安全、亲子互动、自然体验 | 价格敏感度低，决策者通常是妈妈 |
| 👴 退休中老年 | 55+ | 时间自由，偏好慢节奏 | 同龄人结伴、低成本、休闲出行 | 手机操作弱，可能需子女代下单 |

### 3.2 四种运营角色

| 角色 | 权限范围 | 核心职责 |
|:-----|:---------|:---------|
| 平台管理员 | 全局管控 | 钓场维护、商家管理、全局数据、质量管控 |
| 商家管理员 | 本店范围 | 线路创建、团期排期、订单处理、装备管理、数据统计 |
| 领队 | 本团期范围 | 带团出行、查看名单、扫码核销、标记出发/返程 |
| 钓场管理员（V2） | 本钓场范围 | 维护钓场信息、查看预约统计 |

## 五、业务流程

### 5.1 用户预约流程

#### 新用户首次旅程

```
① 打开小程序 → 微信登录授权
② 进入平台发现页 → 看到附近门店列表
③ 点击某门店 → 门店详情（老板介绍、本周活动、往期精彩）
④ 点击某活动 → 活动详情（行程、包含、集合地、取消政策）
⑤ 点击预约 → 选日期 → 选人数 → 勾选装备 → 填联系信息
⑥ 确认支付 → 微信支付 → 获得核销码
⑦ 自动成为该门店"去过的店"
```

#### 老用户复购旅程

```
① 打开小程序 → 自动进入主场店首页
② 看到本周活动 → 有想参加的
③ 点击活动 → 确认详情
④ 一键预约（信息自动填充）→ 支付
⑤ 全程 < 2 分钟
```

### 5.2 商家管理流程

```
① 登录管理后台 → 数据看板（今日订单、营收、待核销）
② 创建/编辑线路 → 关联钓场、设置价格、行程安排
③ 创建团期 → 选日期、设名额、指派领队（支持批量创建）
④ 处理订单 → 查看报名情况、审核退款
⑤ 出行当天 → 领队扫码核销、标记出发/返程
⑥ 出行结束 → 订单自动完成、查看财务统计
```

### 5.3 订单状态机

```
         支付成功           出行当天           核销完成
PENDING ───────→ PAID ───────→ EN_ROUTE ───────→ COMPLETED
  │                │                                    ↑
  │ 超时/取消       │ 申请退款                            │ 团期结束自动完成
  ↓                ↓                                    │
CANCELLED    REFUNDING ───→ REFUNDED              ─────┘
```

## 六、竞品分析与差异化定位

### 6.1 竞品对比

| 维度 | 美团/大众点评 | 携程/飞猪 | 野友网 | **渔乐出行** |
|:-----|:-------------|:----------|:-------|:-------------|
| 定位 | 本地生活综合平台 | 旅行预订平台 | 户外活动社区 | 垂钓专属出行平台 |
| 核心单元 | 商户/门店 | 酒店/机票 | 活动/路线 | 门店（信任单元） |
| 信任机制 | 平台评分 | 平台背书 | 社区口碑 | 老板个人IP+实体门店 |
| 用户关系 | 匿名消费者 | 匿名消费者 | 社区会员 | 门店会员（有主场店） |
| 复购驱动 | 优惠券/红包 | 积分/会员 | 社区粘性 | 私域+会员折扣+裂变 |
| 垂钓专业度 | 低 | 低 | 中 | **高**（钓场+鱼种+装备一体化） |

### 6.2 核心差异化

1. **门店即信任**：用户认识老板、去过店、有归属感，不是匿名平台消费
2. **垂钓专属**：钓场信息（鱼种、设施、钓位）、装备租赁、专业领队一体化
3. **私域+平台双引擎**：80% 复购来自门店老客，20% 新客来自平台发现
4. **实体背书**：线下渔具店实体存在，非纯线上平台，信任成本极低
5. **高频+高复购**：周末固定发团，目标复购率 >60%

## 八、与 cloud-mvp 的关系说明

本项目存在 **两个并行版本**，服务于不同阶段的目标：

| 维度 | cloud-mvp（云开发 MVP） | yule-go（自研版本） |
|:-----|:------------------------|:-------------------|
| **定位** | 快速验证，最小可行产品 | 正式产品版，长期维护 |
| **技术栈** | 微信云函数 + 云数据库 | Go (Gin) + MySQL + Redis |
| **部署** | 微信云开发（零运维） | 需要服务器（Docker） |
| **管理后台** | 小程序内嵌管理页 | Vue3 + Element Plus 独立 PC 端 |
| **开发速度** | 快（1-2 周可上线） | 慢（需 4-6 周） |
| **灵活性** | 受限于云开发能力 | 完全自主可控 |
| **适用场景** | 验证商业模式、收集用户反馈 | 正式运营、多门店扩展 |

**策略**：cloud-mvp 先行验证商业模式和用户需求，yule-go 同步开发正式版本。验证成功后，用户和数据可迁移到 yule-go。

### 2.2 分层职责

| 层级 | 职责 | 依赖 |
|:---|:---|:---|
| Handler | 入参校验、响应封装、调用 Service | Service |
| Service | 业务逻辑、事务管理、调用 Repository | Repository |
| Repository | 数据库 CRUD、SQL 构建 | Model |
| Model | 数据结构定义、GORM 标签 | 无 |

### 2.3 统一响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

错误响应：
```json
{
  "code": 40001,
  "message": "团期已满",
  "data": null
}
```

## 6. 安全设计

| 维度 | 措施 |
|:---|:---|
| 认证 | 微信登录获取 openid，JWT 签发 token |
| 授权 | 管理后台基于角色的访问控制（RBAC） |
| 传输 | 全站 HTTPS，微信支付 V3 |
| 存储 | 敏感数据（手机号）加密存储 |
| 接口 | 频率限制、参数校验、SQL 注入防护 |

# Task 0018 — 消息通知

## 用户故事
作为用户，我希望在订单状态变更时收到微信通知（预约成功、天气变更、出行提醒等）。

## 验收标准
- [ ] AC-01: 微信订阅消息接入（申请模板 + 获取 access_token + 发送消息）
- [ ] AC-02: 预约成功通知（支付成功后推送）
- [ ] AC-03: 出行前提醒（出行前 2 小时推送）
- [ ] AC-04: 天气变更通知（管理员标记天气异常时批量推送）
- [ ] AC-05: 退款结果通知（退款审核通过/驳回时推送）
- [ ] AC-06: 核销成功通知（核销后推送，可选）

## 技术要点
- 微信订阅消息需用户授权（每次授权只能发一条）
- 需要申请消息模板（一次性订阅 / 长期订阅）
- access_token 需要缓存（2 小时有效）
- 批量通知需控制频率（避免被微信限流）

## 依赖
- 0011-微信登录（openid）
- 0012-微信支付（支付成功触发）

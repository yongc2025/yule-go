# 0040 活动管理补全

> **分支**：cloud-mvp（微信云开发）
> **需求源**：`cloud-mvp/REQUIREMENTS.md` §4.2 活动详情页、§5.1 activities 集合、§6.2 activities 云函数

## 背景

当前活动创建/编辑表单仅支持 5 个字段（名称、类型、成人价、儿童价、名额），但活动详情页（用户端）展示了 11 个内容区域。大量内容（图片、亮点、行程、包含/不含、须知）只能通过 seed 数据预填，店主无法通过界面维护。

**目标**：让活动管理界面与详情页一一对应，店主可通过界面录入和修改完整信息。

## 用户故事
作为店主，我需要通过管理界面完整地创建和编辑活动信息，包括图片、行程、费用说明等，让顾客在详情页看到完整、准确的活动介绍。

## 验收标准
- AC-01: 活动创建/编辑支持上传封面图片（最多 9 张，微信云存储）
- AC-02: 活动创建/编辑支持添加亮点标签（预设标签 + 自定义输入）
- AC-03: 活动创建/编辑支持编辑行程安排（时间线：时间 + 描述，可增删）
- AC-04: 活动创建/编辑支持编辑费用包含/不包含（列表项，可增删）
- AC-05: 活动创建/编辑支持填写出行须知（多行文本）
- AC-06: 编辑时回显所有已保存字段，与详情页展示一一对应
- AC-07: 新增字段向后兼容（已有活动数据不受影响，新字段为空时详情页优雅降级）

## 数据模型（activities 集合完整字段）

> 来源：`cloud-mvp/REQUIREMENTS.md` §5.1 activities 集合

| 字段 | 类型 | 当前表单 | 详情页展示 | 本次需补 |
|---|---|---|---|---|
| name | string | ✅ | ✅ 活动名 | - |
| type | string | ✅ | ✅ 类型标签 | - |
| price | number | ✅ | ✅ 价格 | - |
| childPrice | number | ✅ | - | - |
| maxSlots | number | ✅ | - | - |
| images | array[string] | ❌ | ✅ 封面轮播 | **本次** |
| highlights | array[string] | ❌ | ✅ 亮点标签 | **本次** |
| itinerary | array[{time,desc}] | ❌ | ✅ 行程安排 | **本次** |
| includes | array[string] | ❌ | ✅ 费用包含 | **本次** |
| excludes | array[string] | ❌ | ✅ 费用不含 | **本次** |
| notes | string | ❌ | ✅ 出行须知 | **本次** |
| coverImage | string | ❌ | - | **本次**（首张图） |
| merchantId | string | ❌ | ✅ 门店名 | 暂不改（MVP 单店） |
| status | string | - | - | - |

## 约束
- 图片上传使用微信云存储（wx.cloud.uploadFile）
- 新字段向后兼容：已有活动的新字段为空时，详情页需优雅隐藏对应区域
- 表单从弹窗升级为独立页面（字段太多，弹窗放不下）

# Task 0012 — 微信支付

## 用户故事
作为小程序用户，我需要通过微信支付完成订单付款和会员充值。

## 验收标准
- [ ] AC-01: 订单创建后调用微信支付统一下单 API，返回支付参数
- [ ] AC-02: 小程序调用 `wx.requestPayment()` 完成支付
- [ ] AC-03: 微信支付异步回调验签（RSA V3 签名），更新订单状态
- [ ] AC-04: 支付超时自动关闭（已有定时任务，需同步关闭微信侧）
- [ ] AC-05: 支付掉单补单（主动查询微信支付订单状态）
- [ ] AC-06: 会员充值支付（统一下单 + 回调，更新余额 + 会员等级）
- [ ] AC-07: 退款接口（管理员审核通过 → 调用微信退款 API）
- [ ] AC-08: 退款回调（更新退款状态）
- [ ] AC-09: 余额 + 微信混合支付（余额抵扣一部分，微信支付剩余）

## 技术要点
- 微信支付 V3 API 需要 RSA 私钥签名
- 需要商户号、API 证书、APIv3 密钥
- 回调需验签（微信平台证书）
- 退款需要商户号退款权限
- 金额单位：微信用"分"，系统用"元"，需转换

## 依赖
- 0011-微信登录（用户身份）
- 需要微信支付商户号（企业资质）

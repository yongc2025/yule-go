# Task 0013 — 钓场钓位体系

## 用户故事
作为运营者，我需要管理多个钓场和钓位；作为用户，我需要浏览钓场、选择钓位来预约。

## 验收标准
- [ ] AC-01: `fishing_spots` 表创建，管理后台 CRUD
- [ ] AC-02: `spot_grades` 表创建，管理后台管理钓位等级
- [ ] AC-03: `fishing_positions` 表创建，管理后台批量创建钓位
- [ ] AC-04: `schedule_positions` 表创建，团期创建时自动生成钓位库存
- [ ] AC-05: `routes` 表新增 `fishing_spot_id` 字段
- [ ] AC-06: 管理后台 — 钓场管理页面（列表/详情/编辑/上下架）
- [ ] AC-07: 管理后台 — 钓位管理页面（等级管理/钓位列表/批量创建）
- [ ] AC-08: 小程序 — 钓场列表页（按距离排序，需位置授权）
- [ ] AC-09: 小程序 — 钓场详情页（设施/鱼种/图片/关联团期）
- [ ] AC-10: 小程序 — 钓位选择页（按等级分组，实时占用状态）
- [ ] AC-11: 预约下单时关联钓位，创建 `schedule_positions` 记录
- [ ] AC-12: 取消订单时释放钓位库存

## 技术要点
- 钓位并发预约需 SELECT FOR UPDATE 或 Redis 分布式锁
- 团期创建时批量插入 schedule_positions（该钓场所有钓位）
- 附近钓场可用 Redis GEO 或 MySQL 空间函数
- 钓位选择需要 WebSocket 或轮询实现实时状态

## 依赖
- 无前置依赖（可与 0011 并行）

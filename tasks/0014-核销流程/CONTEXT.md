# Task 0014 — 核销流程

## 用户故事
作为用户，我支付成功后获得核销码，出行当天出示给领队扫码确认到场。
作为领队，我需要扫码/输入核销码确认用户到场，实时查看到场率。

## 验收标准
- [ ] AC-01: 支付成功后自动生成核销码（6-8位数字）+ 二维码数据
- [ ] AC-02: `checkins` 表创建
- [ ] AC-03: 小程序"我的订单"展示核销二维码
- [ ] AC-04: 管理后台核销页面（选择团期 → 扫码/手动输入）
- [ ] AC-05: 核销时校验（订单状态、是否已核销、是否在有效期内）
- [ ] AC-06: 核销后更新订单状态 + checkins 记录
- [ ] AC-07: 实时到场统计（已核销/总人数）
- [ ] AC-08: 团期结束后核销码自动过期

## 技术要点
- 核销码生成：HMAC-SHA256(order_no + timestamp, secret) 截取 6-8 位
- 二维码内容：加密后的核销码 + 订单号（JSON 格式）
- 扫码核销：小程序端展示二维码，管理后台用扫码枪或摄像头扫描
- GPS 辅助校验（可选）：校验用户是否在钓场范围内

# CONTEXT — 0008 定时任务调度

## 问题来源
审计发现 S8：超时订单自动取消逻辑已在 repository 实现，但未接入定时调度。

## 验收标准
- AC-01：每分钟检查一次超时未支付订单（15 分钟）
- AC-02：自动取消超时订单，释放名额 + 退还库存 + 退还余额
- AC-03：支持手动触发（管理后台按钮）
- AC-04：调度器优雅退出（收到信号时等待当前任务完成）

## 技术方案
- 使用 `robfig/cron` 库实现 cron 调度
- 在 main.go 中启动调度器
- 提供管理后台 API 手动触发

## 优先级
P0 — 上线前必须完成

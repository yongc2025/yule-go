# CONTEXT — 0001 团期管理

## 需求摘要

为管理后台提供团期的创建、编辑、取消功能；为小程序端提供团期列表展示（按周查看）。

## 用户故事

**后台管理员：** 我想每周提前创建下周的团期（选择线路、日期、名额、领队），以便客户可以提前报名。

**小程序用户：** 我想查看本周和下周有哪些团可以报名，还有多少名额，以便选择合适的团期。

## 验收标准

- AC-01：管理员可以在后台创建团期，选择线路、出行日期、最大名额、领队信息
- AC-02：管理员可以编辑已创建的团期（修改名额、领队等）
- AC-03：管理员可以取消团期（已报名的客户自动通知）
- AC-04：小程序端可以按周查看团期列表，显示线路名、日期、剩余名额
- AC-05：名额满时自动标记为"已满"，不可再报名
- AC-06：同一个线路同一天不能创建重复团期

## 约束

- 团期日期只能是周六或周日
- 创建团期时自动校验线路是否存在且已上架
- 取消团期时如果有已支付订单，需触发退款流程

## 依赖

- 数据库：routes 表、schedules 表
- API：线路管理接口（查询线路详情）
- 微信消息：团期变更通知（V2 实现，MVP 先不做）

## 技术决策

- 团期状态机：报名中 → 已满 → 已出发 → 已完成（支持直接取消）
- 已报名人数通过 orders 表实时统计（不冗余存储，避免数据不一致）

# 0039 退款资金对账

> **分支**：cloud-mvp（微信云开发）
> **需求源**：`cloud-mvp/REQUIREMENTS.md` §7.2 费用计算、§7.5 取消退款政策

## 用户故事
作为店主/运营，需要确保每笔退款金额正确、可追溯，上线前资金链路无误。

## 验收标准
- AC-01: 每笔退款记录存储在 orders 集合（refundAmount, refundAt, refundNo）
- AC-02: admin 后台可查看退款订单列表
- AC-03: admin 后台展示退款统计（今日退款笔数、金额）
- AC-04: 退款状态与微信支付后台一致
- AC-05: 模拟支付模式下退款标记正确

## 对账维度
| 维度 | 说明 |
|---|---|
| 按日 | 每日支付总额 vs 退款总额 = 净收入 |
| 按团期 | 每个团期的收入/退款/净收入 |
| 按订单 | 每笔订单的支付/退款明细 |

## 费用计算规则（cloud-mvp/REQUIREMENTS.md §7.2）

- 成人费用 = 活动价格 × 成人数
- 儿童费用 = 儿童价 × 儿童数
- 总费用 = 成人费用 + 儿童费用
- 退款金额 ≤ 原订单总费用

## 约束
- 真实支付退款需微信支付商户证书
- 对账数据 T+1 可用（微信退款有延迟）
- MVP 阶段全额退款，对账逻辑需预留时间梯度扩展

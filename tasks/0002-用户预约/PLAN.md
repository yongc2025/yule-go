# PLAN — 0002 用户预约

## 执行计划

### 阶段 1：数据库层
1. 创建 orders、order_rentals 表迁移脚本
2. 定义 Order、OrderRental Model
3. 实现 OrderRepository

### 阶段 2：后端 API
1. 实现价格计算 Service（团费 + 租赁费 - 折扣 - 余额）
2. 实现订单创建 Service（含名额锁定、超时释放）
3. 实现微信支付集成（下单、回调）
4. 实现订单查询/取消 API

### 阶段 3：小程序前端
1. 预约页面（选线路 → 填信息 → 选租赁 → 确认支付）
2. 支付调用
3. 订单详情页
4. 订单列表页

## 验证步骤

1. 创建订单 → 返回支付参数
2. 模拟支付回调 → 订单状态变更为已支付
3. 15 分钟不支付 → 订单自动取消
4. 名额满时下单 → 返回错误
5. 会员下单 → 自动计算折扣

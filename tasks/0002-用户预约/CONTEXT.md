# CONTEXT — 0002 用户预约

## 需求摘要

小程序端完整预约流程：选线路 → 选团期 → 填写人数/联系信息 → 选择租赁项 → 选择是否用余额/会员折扣 → 微信支付 → 生成订单。

## 用户故事

**小程序用户：** 我想在小程序上选择一个周末的钓鱼团，填写出行人数和联系方式，支付后收到确认，到时候直接去渔具店集合就行。

## 验收标准

- AC-01：用户选择线路后，能看到该线路近期的团期和剩余名额
- AC-02：用户可以填写成人/儿童人数，系统自动计算团费
- AC-03：用户可以勾选装备租赁项，费用实时更新
- AC-04：会员用户自动享受折扣，可选择是否用余额抵扣
- AC-05：支付成功后生成订单，团期已报名人数 +N
- AC-06：支付超时（15 分钟）自动释放名额
- AC-07：同一用户同一团期不能重复下单（已取消的除外）

## 约束

- 支付必须通过微信支付，不支持其他方式
- 儿童价格仅对亲子团有效，其他线路儿童按成人计
- 订单创建后 15 分钟内未支付自动取消

## 依赖

- 数据库：orders、order_rentals、schedules、users 表
- 外部服务：微信支付 V3 API
- 裂变模块：检查是否有邀请码、计算新客立减

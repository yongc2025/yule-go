# Task 0015 — 订单状态机重构

## 用户故事
作为系统，我需要完整的订单状态流转，覆盖退款、核销、自动完成等场景。

## 验收标准
- [ ] AC-01: 订单状态扩展为 8 个：PENDING/PAID/EN_ROUTE/CHECKIN/COMPLETED/CANCELLED/REFUNDING/REFUNDED
- [ ] AC-02: 退款规则：出行前72h+全退、48-72h退80%、24-48h退50%、24h内不退
- [ ] AC-03: 天气/不可抗力取消 → 100% 全退
- [ ] AC-04: 订单状态机校验（非法状态转换拒绝）
- [ ] AC-05: 定时任务：团期次日自动将"已出行"订单变为"已完成"
- [ ] AC-06: 管理后台退款审核（通过/驳回 + 原因）
- [ ] AC-07: 退款金额计算（根据取消时间和退款规则）
- [ ] AC-08: 退款后释放钓位库存 + 退还租赁库存 + 退还余额

## 技术要点
- 状态机校验用 map[OrderStatus][]OrderStatus 定义合法转换
- 退款规则可配置化（写入 config 或数据库）
- 自动完成定时任务需关联团期状态

## 依赖
- 0013-钓场钓位体系（钓位库存释放）
- 0012-微信支付（退款 API 调用）

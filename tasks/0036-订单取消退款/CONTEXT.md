# 0036 用户取消订单 + 退款规则

> **分支**：cloud-mvp（微信云开发）
> **需求源**：`cloud-mvp/REQUIREMENTS.md` §7.5 取消退款政策、§4.4 订单状态

## 用户故事
作为用户，我支付后如果需要取消订单，可以申请退款。退款金额原路返回到微信钱包。

## 验收标准
- AC-01: 用户可在订单详情页取消已支付订单（团期前任意时间）
- AC-02: 取消后调用微信支付退款接口，金额原路返回
- AC-03: 订单状态变更为 cancelled，名额自动归还（bookedSlots -1）
- AC-04: 店主可在后台看到退款记录
- AC-05: 退款比例配置化（当前默认 100%，代码预留比例配置结构）

## 退款规则

### 需求定义（cloud-mvp/REQUIREMENTS.md §7.5）

| 取消时间 | 退款比例 |
|:---|:---|
| 出行前 72 小时+ | 100% 全退 |
| 出行前 24-72 小时 | 50% |
| 出行前 24 小时内 | 不可退款 |
| 天气/不可抗力 | 100% 全退 |

### MVP 阶段简化

> 需求文档明确：**"MVP 阶段简化处理：取消订单直接全额退，退款审核线下处理。"**

| 场景 | 退款比例 | 说明 |
|---|---|---|
| 用户主动取消 | 100% | MVP 简化，代码预留时间梯度配置结构 |
| 店主取消团期 | 100% | 自动触发全额退 |
| 不可抗力取消 | 100% | 线下审核 |

> 退款比例通过 `refundPolicy` 配置对象管理，V2 接入时间梯度逻辑

## 订单状态机（cloud-mvp/REQUIREMENTS.md §5.1 orders）

```
         支付成功              核销完成
PENDING ───────→ PAID ───────→ COMPLETED
  │                                ↑
  │ 取消                           │
  ↓                                │
CANCELLED                     团期结束自动完成
```

## 约束
- 退款需微信支付商户号 + API 证书
- 模拟支付模式下退款也模拟处理
- 退款金额 ≤ 原订单金额

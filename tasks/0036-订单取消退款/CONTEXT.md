# 0036 用户取消订单 + 退款规则

## 用户故事
作为用户，我支付后如果需要取消订单，可以申请退款。退款金额原路返回到微信钱包。

## 验收标准
- AC-01: 用户可在订单详情页取消已支付订单（团期前任意时间）
- AC-02: 取消后调用微信支付退款接口，金额原路返回
- AC-03: 订单状态变更为 cancelled，名额自动归还
- AC-04: 店主可在后台看到退款记录
- AC-05: 退款比例配置化（当前默认 100%，代码预留比例配置结构）

## 退款规则（当前配置）
| 场景 | 退款比例 |
|---|---|
| 用户主动取消 | 100% |
| 店主取消团期 | 100% |
| 不可抗力取消 | 100% |

> 退款比例通过 `refundPolicy` 配置对象管理，后续可根据业务需求调整

## 约束
- 退款需微信支付商户号 + API 证书
- 模拟支付模式下退款也模拟处理
- 退款金额 ≤ 原订单金额

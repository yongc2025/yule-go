## ⚠️ 双分支说明

本项目存在 **两个并行分支**，任务和需求文档需严格区分归属：

| 维度 | 自研版（yule-go） | 云开发 MVP（cloud-mvp） |
|:---|:---|:---|
| 后端 | Go (Gin) | 微信云函数 (Node.js) |
| 数据库 | MySQL（关系型，用"表"） | 云数据库（文档型，用"集合"） |
| 需求源 | `memory-bank/requirements.md` | `cloud-mvp/REQUIREMENTS.md` |
| 管理后台 | Vue3 独立 PC 端 | 小程序内嵌管理页 |
| 任务范围 | V4-01 ~ V4-20, 0001-0034 | 0035-0039 |
| 数据模型术语 | 表 (table)、字段 (column) | 集合 (collection)、文档 (document) |
| 部署 | 需要服务器 | 微信云开发（零运维） |

**规则**：
- 自研版任务的 CONTEXT.md 引用 `memory-bank/requirements.md`
- cloud-mvp 任务的 CONTEXT.md 引用 `cloud-mvp/REQUIREMENTS.md`
- 数据模型术语必须与分支匹配：自研版用"表/字段"，cloud-mvp 用"集合/文档"
- 两个分支的需求变更互不干扰，各自维护

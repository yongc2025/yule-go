# 渔乐出行 — 云开发 MVP 需求规格书

> **版本**：v1.0
> **日期**：2026-05-08
> **定位**：单店最小可用版本，验证核心业务流程
> **技术栈**：微信小程序原生 + 微信云开发（云函数 + 云数据库）

---

## 一、产品定位

依托线下渔具实体店，打造"周末垂钓出行"小程序。用户可以在小程序上浏览本周活动、预约报名、微信支付、获得核销码，到现场出示核销。

**一句话**：让渔具店老客人在手机上一键报名周末钓鱼团。

---

## 二、目标用户

| 角色 | 说明 |
|:---|:---|
| **普通用户** | 渔具店老客人、钓鱼爱好者、亲子家庭。在小程序浏览活动、下单预约 |
| **店主（管理员）** | 渔具店老板。管理活动、查看订单、核销到场 |

---

## 三、核心业务流程

```
用户打开小程序
  │
  ├── 首页：看到门店信息 + 本周活动列表
  │
  ├── 点击某活动 → 活动详情
  │     ├── 行程安排
  │     ├── 费用包含/不包含
  │     ├── 选择档期（日期 + 剩余名额）
  │     └── 取消政策
  │
  ├── 点击"立即报名" → 预约下单页
  │     ├── 选择人数（成人/儿童）
  │     ├── 填写联系信息
  │     └── 确认支付 → 微信支付
  │
  ├── 支付成功 → 获得核销码（6 位数字 + 二维码）
  │
  └── 出行当天 → 出示核销码 → 店主扫码核销 → 完成
```

## 四、页面功能说明

### 4.1 首页（活动列表）

**入口**：小程序启动默认页

| 区域 | 内容 | 说明 |
|:---|:---|:---|
| 门店头部 | 店主头像 + 门店名 + 口号 + 评分 + 已服务人数 | 建立信任感 |
| 周选择器 | ◀ 本周 ▶ 左右切换查看不同周的活动 | 支持上周/本周/下周 |
| 活动卡片列表 | 封面图 + 活动名 + 类型标签 + 日期 + 剩余名额 + 价格 | 卡片式布局，点击进入详情 |

**交互逻辑**：
- 默认展示本周活动
- 左右箭头切换周
- 点击"本周"文字回到当前周
- 下拉刷新
- 名额 ≤ 3 位时显示橙色警告
- 已满的活动显示"已满"标签，报名按钮置灰

### 4.2 活动详情页

**入口**：首页点击活动卡片

| 区域 | 内容 | 说明 |
|:---|:---|:---|
| 封面轮播 | 多张活动实拍图 | 自动轮播 |
| 基本信息 | 活动名 + 所属门店 + 价格 + 亮点标签 | 标签如"含午餐""新手友好" |
| 行程安排 | 时间线形式 | 如"08:00 集合出发 → 09:30 到达钓点..." |
| 费用包含 | ✓ 列表 | 往返车费、专属钓点、基础饵料等 |
| 费用不含 | ✗ 列表 | 个人消费、额外渔具等 |
| 档期选择 | 可选日期 + 剩余名额 | 点击选中，已满不可选 |
| 出行须知 | 注意事项文本 | 如"请穿运动鞋、带防晒" |
| 取消政策 | 退改规则 | 72h+全退 / 24-72h退50% / 24h内不退 |
| 底部固定栏 | 价格 + "立即报名"按钮 | 未选档期时提示"请先选择档期" |

### 4.3 预约下单页

**入口**：活动详情页点击"立即报名"

| 区域 | 内容 | 说明 |
|:---|:---|:---|
| 活动摘要 | 活动名 + 档期日期 + 门店名 | 从上一页传入 |
| 人数选择 | 成人 ± / 儿童 ± | 成人最少 1 位，显示单价 |
| 联系信息 | 姓名 + 手机号 | 手机号 11 位校验 |
| 备注 | 多行文本（选填） | 最多 200 字 |
| 费用明细 | 团费 + 合计 | 实时计算 |
| 底部固定栏 | 合计金额 + "确认支付" | 点击后调用微信支付 |

**校验规则**：
- 姓名不能为空
- 手机号必须是 11 位有效号码
- 费用二次校验（前端传的总价 vs 后端计算的总价）

**支付流程**：
1. 调用云函数 `orders.create` 创建订单
2. 云函数返回微信支付参数
3. 前端调用 `wx.requestPayment` 发起支付
4. 支付成功 → 跳转订单列表
5. 支付取消 → 提示"已取消支付"

### 4.4 我的订单页

**入口**：底部 TabBar

| 区域 | 内容 | 说明 |
|:---|:---|:---|
| 状态筛选 | 全部 / 已支付 / 已完成 / 已取消 | Tab 切换 |
| 订单卡片 | 订单号 + 状态 + 活动名 + 日期 + 人数 + 金额 | 按创建时间倒序 |

**订单状态与操作**：

| 状态 | 显示 | 可操作 |
|:---|:---|:---|
| 待支付 (pending) | 橙色"待支付" | 取消订单 |
| 已支付 (paid) | 蓝色"已支付" | 出示核销码 |
| 已完成 (completed) | 绿色"已完成" | 无 |
| 已取消 (cancelled) | 灰色"已取消" | 无 |

**核销码展示**（已支付订单）：
- 6 位数字核销码（大号绿色字体）
- 二维码图片
- 出行当天出示给领队/店主

### 4.5 管理入口页

**入口**：首页底部或个人中心

**登录**：
- 输入管理密码（MVP 简化版，密码存在云数据库 `admins` 集合）
- 验证通过进入管理面板

**管理面板内容**：

| 区域 | 内容 |
|:---|:---|
| 今日概览 | 今日订单数、今日营收、待核销人数、本周团期数 |
| 扫码核销入口 | 点击进入扫码页面（TODO） |
| 今日团期 | 今日的团期列表 + 已报名/已核销统计 |
| 最近订单 | 最近 5 笔订单（联系人 + 活动名 + 金额 + 状态） |

## 六、API 接口（云函数）

### 6.1 login

| 参数 | 说明 |
|:---|:---|
| 入参 | 无（自动获取 openid） |
| 返回 | `{ openid }` |

### 6.2 activities

| action | 入参 | 返回 | 说明 |
|:---|:---|:---|:---|
| `list` | weekStart, weekEnd, page, pageSize | 活动列表 + total | 按周查询 |
| `detail` | id | 活动详情 | 单条查询 |
| `create` | 活动字段 | `{ _id }` | 管理员创建 |
| `update` | id + 更新字段 | 成功/失败 | 管理员编辑 |

### 6.3 orders

| action | 入参 | 返回 | 说明 |
|:---|:---|:---|:---|
| `create` | activityId, scheduleId, adults, children, contactName, contactPhone, remark, totalFee | `{ orderId, orderNo, payment }` | 创建订单 + 发起支付 |
| `payCallback` | outTradeNo, resultCode | 无 | 微信支付回调（自动触发） |
| `cancel` | orderId | 成功/失败 | 取消订单 + 退还名额 |
| `list` | page, pageSize | 订单列表 + total | 用户的订单 |
| `detail` | orderId | 订单详情 | 单条查询 |

### 6.4 checkin

| action | 入参 | 返回 | 说明 |
|:---|:---|:---|:---|
| `scan` / `manual` | checkinCode, scheduleId | 核销结果 + 订单信息 | 核销 |
| `stats` | scheduleId | total, checkedIn, unchecked, checkedList | 团期核销统计 |

## 七、业务规则

### 7.1 名额管理

- 创建订单时扣减团期 `bookedSlots`
- 取消订单时退还 `bookedSlots`
- 名额不足时拒绝下单，提示剩余位数
- 前端实时显示剩余名额（≤3 位时橙色警告）

### 7.2 费用计算

- 成人费用 = 活动价格 × 成人数
- 儿童费用 = 儿童价 × 儿童数（部分活动无儿童价）
- 总费用 = 成人费用 + 儿童费用
- 后端二次校验费用，防止前端篡改

### 7.3 订单超时

- MVP 阶段不做自动超时取消（依赖云函数定时触发器，增加复杂度）
- 待支付订单由用户手动取消
- V2 可加入 15 分钟超时自动取消

### 7.4 核销规则

- 仅已支付（paid）状态的订单可核销
- 同一订单多次核销只生效一次（幂等）
- 核销后订单状态变为 completed
- 仅管理员可执行核销操作

### 7.5 取消退款政策

| 取消时间 | 退款比例 |
|:---|:---|
| 出行前 72 小时+ | 100% 全退 |
| 出行前 24-72 小时 | 50% |
| 出行前 24 小时内 | 不可退款 |
| 天气/不可抗力 | 100% 全退 |

> MVP 阶段简化处理：取消订单直接全额退，退款审核线下处理。

## 八、MVP 范围

### ✅ 包含

- 微信登录（自动获取 openid）
- 活动列表（按周展示）
- 活动详情（行程、包含、档期、取消政策）
- 预约下单（选人数、填信息、微信支付）
- 订单管理（列表、状态、核销码）
- 核销（手动输入核销码）
- 管理面板（今日概览、最近订单）

- 管理后台 PC 端
- 会员充值体系
- 老带新裂变
- 装备租赁
- 退款审核流程
- 批量创建团期
- 数据看板趋势图
- 天气预报
- 分享海报
- 门店视频介绍
- 扫码核销（摄像头扫描）

## 九、部署与配置

### 前置条件

1. 微信公众平台注册小程序（个人主体可注册）
2. 开通云开发（免费额度足够 MVP 使用）
3. 微信开发者工具

### 配置项

| 配置 | 位置 | 说明 |
|:---|:---|:---|
| AppID | `project.config.json` → `appid` | 小程序 AppID |
| 云环境 ID | `miniprogram/app.js` → `env` | 云开发环境 ID |
| 微信支付子商户号 | `cloudfunctions/orders/index.js` → `subMchId` | 如有子商户 |

### 初始化数据

1. 创建 7 个数据库集合
2. `merchants` 集合添加门店信息
3. `admins` 集合添加管理员（openid + password）
4. `activities` 集合添加活动
5. `schedules` 集合添加团期
6. 部署 4 个云函数（右键 → 上传并部署）

## 十、与正式版（yule-go）的关系

| 维度 | 云开发 MVP | 正式版 yule-go |
|:---|:---|:---|
| 定位 | 快速验证需求 | 正式产品 |
| 后端 | 云函数（Node.js） | Go (Gin) |
| 数据库 | 云数据库（文档型） | MySQL（关系型） |
| 管理后台 | 小程序内嵌 | Vue3 独立后台 |
| 扩展性 | 受限于云开发 | 自由扩展 |
| 运维 | 零运维 | 需要服务器 |

**数据迁移路径**：MVP 验证通过后，将云数据库数据导出，导入 MySQL，小程序前端切换 API 地址指向 Go 后端。

## 二、逐项详细分析

### 差异 1：整体色彩体系 🔴

**设计稿**：
```css
--primary: #2D6A4F;        /* 森林绿 — 自然、信任 */
--primary-light: #40916C;
--primary-lighter: #52B788;
--primary-bg: #D8F3DC;
--accent: #E76F51;          /* 暖橙 — 活力、行动 */
--accent-light: #F4A261;
--bg-page: #F5F5F0;         /* 暖米色背景 */
```

**实际实现**：
```css
/* 主色蓝色 #1890ff，价格红色 #f5222d，背景灰色 #f5f5f5 */
```

**影响**：整体视觉风格完全偏离设计定位（"自然、温暖、信任"），变成了通用蓝色模板。

---

### 差异 2：TabBar 导航 🔴

**设计稿**：4 个 Tab
| Tab | 图标 | 页面 |
|:----|:-----|:-----|
| 首页 | 🏠 | 活动列表 |
| 发现 | 🔍 | 附近门店 |
| 订单 | 📋 | 我的订单 |
| 我的 | 👤 | 个人中心 |

**实际实现**：2 个 Tab
| Tab | 页面 |
|:----|:-----|
| 活动 | 活动列表 |
| 订单 | 我的订单 |

**缺失**：「发现」和「我的」两个 Tab 及对应页面。

---

### 差异 3：发现页（附近门店）🟡

**设计稿功能**：
- 搜索门店或活动
- 分类筛选（全部/钓鱼/露营/亲子/慢游）
- 门店卡片（头像、评分、已服务人数、距离、本周活动数、热门活动、价格）
- 点击进入门店详情

**MVP 需求关联**：REQUIREMENTS.md 中「用户门店关系」是 MVP 范围，需要有发现门店的入口。

---

### 差异 4：我的（个人中心）🟡

**设计稿功能**：
- 用户头像 + 昵称 + 会员等级 + 余额
- 我的门店（去过的店，主/场店标记）
- 快捷入口：我的订单、会员充值、邀请好友、联系客服、发现新门店

**MVP 需求关联**：「用户门店关系」需要展示，但会员充值/邀请好友是 V2。

---

### 差异 5：首页门店头部 🟡

**设计稿**：
- 店主头像 + 店名 + 营业状态
- 评分 + 已服务人数 + 好评率 + 开店时长
- 地址（含距离）+ 电话 + 营业时间
- 导航/拨打按钮
- 视频介绍入口（▶️ 听老王策）

**实际实现**：
- 店主头像 + 店名 + 口号
- 评分 + 已服务人数

**缺失**：好评率、开店时长、地址电话、营业时间、导航按钮、视频入口。

---

### 差异 6：活动卡片图标 🟢

**设计稿**：每种活动类型有对应图标
| 类型 | 图标 |
|:-----|:-----|
| fishing | 🎣 |
| camping | ⛺ |
| family | 👨👩👧 |
| senior | 👴 |
| wild_fishing | 🎣 |

**实际实现**：仅有文字标签（"钓鱼团"、"露营套餐"等），无图标。

---

### 差异 7：活动详情 — 天气/钓场 🟢

**设计稿**：
- 天气预报模块（明日天气 + 建议穿搭）
- 关联钓场模块（钓场名、鱼种、停车位、距离、车程）

**MVP 需求**：REQUIREMENTS.md 明确将「天气预报」列为 V2。钓场信息不在 MVP 范围。

---

### 差异 8：活动详情 — 集合地点 🟢

**设计稿**：
- 独立「集合地点」模块
- 门店名 + 地址
- 🧭 导航按钮

**实际实现**：无此模块。

**建议**：MVP 可加简单版本（纯文本展示集合地点），增强用户体验。

---

### 差异 9：预约页 — 装备租赁 🟡

**设计稿**：
- 装备租赁区域（鱼竿套装 ¥30、折叠桌椅 ¥25、天幕 ¥40）
- 可勾选，自动加入费用明细
- 会员折扣自动计算

**MVP 需求**：REQUIREMENTS.md 明确将「装备租赁」列为 V2。

---

### 差异 10：订单页 — 二维码展示 🟡

**设计稿**：
- 6 位数字核销码（大号绿色字体）
- 二维码图片（可扫描）

**实际实现**：
- 6 位数字核销码 ✅
- canvas 占位但未实际生成二维码 ❌

**影响**：核销流程依赖二维码扫描（设计稿），当前仅有手动输入核销码。

---

### 差异 11：门店详情页 🟡

**设计稿**（02-store-detail.html）：
- 门店完整信息（头像、店名、老板、评分详情）
- 统计数据（已服务/评分/好评率/开店时长）
- 地址/电话/营业时间 + 导航/拨打
- 本周活动列表
- 用户评价模块

**实际实现**：不存在独立门店详情页。

**MVP 需求**：门店详情不在明确的 MVP 列表中，但「用户门店关系」需要门店信息展示。

---

## 三、优先级分类

### 🔴 P0 — 必须修复（MVP 核心差异）

1. **色彩体系对齐**：将全局主色从蓝色改为森林绿，背景改为暖米色
2. **TabBar 扩展**：从 2 Tab 扩展为 4 Tab（首页/发现/订单/我的）
3. **订单二维码**：实现核销码的二维码生成与展示

### 🟡 P1 — 建议修复（提升设计还原度）

4. **首页门店头部增强**：补充地址/电话/营业时间/好评率/导航按钮
5. **活动卡片图标**：为每种活动类型添加图标
6. **发现页（简化版）**：门店列表 + 分类筛选（不含搜索，MVP 简化）
7. **我的页（简化版）**：用户信息 + 我的门店 + 订单入口（不含会员/充值）
8. **活动详情 — 集合地点**：添加简单的集合地点展示

### 🟢 P2 — 可选优化（V2 预留）

9. 活动详情 — 天气预报（V2）
10. 活动详情 — 关联钓场（V2）
11. 预约页 — 装备租赁（V2）
12. 门店详情独立页（V2）

---

## 四、结论

设计稿与实际实现之间存在 **系统性偏差**，主要体现在：

1. **色彩体系完全不同** — 设计稿是自然绿色调，实现是通用蓝色调
2. **导航结构缺失** — 4 Tab 变成了 2 Tab，缺失发现和个人中心
3. **多个页面/模块未实现** — 发现页、我的页、门店详情页、集合地点等
4. **关键功能不完整** — 订单核销码缺少二维码生成

建议按照 P0 → P1 → P2 的优先级逐步调整，使实现对齐设计稿。

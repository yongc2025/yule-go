## 一、版本规划总览

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   MVP       │    │    V2       │    │    V3       │
│  已完成 ✅   │───→│  规划中     │───→│  远期目标    │
│             │    │             │    │             │
│ 核心业务验证 │    │ 功能完善     │    │ 规模化运营   │
│ 单店验证     │    │ 多店扩展     │    │ 平台化      │
└─────────────┘    └─────────────┘    └─────────────┘
```

| 版本 | 定位 | 核心目标 | 状态 |
|:---|:---|:---|:---|
| **MVP** | 单店最小可用版本 | 验证核心流程：活动展示→预约支付→核销 | ✅ 已完成 |
| **V2** | 功能完善版本 | 补全支付、超时、管理后台、批量操作 | 📋 规划中 |
| **V3** | 规模化版本 | 迁移至 Go 后端 + MySQL，支持多店、数据分析 | 🔮 远期 |

## 三、MVP 待完善项

> 基于 yule-go 审计报告和 cloud-mvp 代码审查，提取需要在 V2 阶段修复的 P0/P1 问题。

### 3.1 P0 — 必须修复

| # | 问题 | 模块 | 修复方案 | 预估工时 |
|:---|:---|:---|:---|:---|
| P0-1 | **模拟支付未替换** | orders | 配置微信支付子商户号，接入真实支付 | 1 天 |
| P0-2 | **待支付订单无超时取消** | orders | 使用云函数定时触发器，15 分钟超时自动取消 + 退还名额 | 1 天 |
| P0-3 | **管理员密码明文存储** | admins | 使用 bcrypt 或云函数内置加密存储密码 | 0.5 天 |

### 3.2 P1 — 建议修复

| # | 问题 | 模块 | 修复方案 | 预估工时 |
|:---|:---|:---|:---|:---|
| P1-1 | **订单创建非原子操作** | orders | 涉及 schedules/coupons/user_wallets/orders 四个集合写入，任一失败无法回滚 | 0.5 天 |
| P1-2 | **核销码冲突风险** | orders | 6 位随机码理论冲突概率低但无唯一校验，建议增加唯一索引或扩展位数 | 0.5 天 |
| P1-3 | **优惠券类型硬编码** | coupons | 券类型配置散落在多个云函数中，应统一到配置集合 | 0.5 天 |
| P1-4 | **无请求频率限制** | 所有云函数 | 云函数层面无防刷机制，建议增加简单频率限制 | 1 天 |
| P1-5 | **错误日志不够详细** | 所有云函数 | 部分 catch 块仅打印 message，缺少堆栈和上下文信息 | 0.5 天 |
| P1-6 | **用户删除相册后云存储文件未清理** | users | deleteAlbum 仅删除数据库记录，云存储文件残留 | 0.5 天 |

## 七、与 yule-go 的协作关系

| 维度 | cloud-mvp | yule-go |
|:---|:---|:---|
| **定位** | 快速验证 MVP | 正式产品 |
| **后端** | 云函数（Node.js） | Go (Gin) |
| **数据库** | 云数据库（文档型） | MySQL（关系型） |
| **管理后台** | 小程序内嵌 | Vue3 独立后台 |
| **扩展性** | 受限于云开发 | 自由扩展 |
| **运维** | 零运维 | 需要服务器 |
| **生命周期** | 验证完可迁移 | 长期维护 |

# 渔乐出行 — 产品介绍文档

> **cloud-mvp（微信云开发版本）**
> **版本**：v1.0 | **日期**：2026-05-08

---

## 一、产品愿景

**一句话**：让渔具店老客人在手机上一键报名周末钓鱼团。

**详细描述**：渔乐出行是一个以线下渔具实体店为载体的周末短途旅行数字化运营系统。我们帮助渔具店老板将"周末带客人钓鱼"这件事从口头通知、微信群接龙，升级为线上预约、微信支付、到场核销的标准化流程。用户打开小程序，浏览本周活动，选择档期，一键下单，出行当天出示核销码即可——简单、高效、可追溯。

## 二、目标用户画像

### 2.1 普通用户（C 端）

- **人群**：渔具店老客人、钓鱼爱好者、亲子家庭
- **年龄**：28-55 岁为主
- **场景**：周末想出去钓鱼，不想操心路线、钓点、交通
- **痛点**：
  - 微信群接龙容易遗漏，不确定是否报名成功
  - 不知道这周有什么活动、还有没有名额
  - 现金支付不方便，转账记录不好查
- **核心需求**：一键报名、在线支付、查看订单和核销码

### 2.2 店主 / 管理员（B 端）

- **人群**：渔具店老板、领队
- **场景**：每周组织 1-3 次钓鱼团活动，管理报名、收费、核销
- **痛点**：
  - 微信群统计人数费时费力，容易出错
  - 收款靠转账，对账麻烦
  - 不清楚哪些人到场了、哪些没来
- **核心需求**：发布活动、查看订单、扫码/手动核销、营收统计

### 2.3 合作伙伴 / 投资人

- **关注点**：商业模式验证、用户增长数据、技术可行性
- **核心需求**：快速了解产品逻辑、当前状态、下一步规划

## 三、核心功能概览

### 3.1 用户端（小程序）

| 功能模块 | 说明 |
|:---|:---|
| **首页（活动列表）** | 门店信息展示 + 按周浏览活动卡片 + 名额实时显示 |
| **活动详情** | 封面轮播、行程安排、费用包含/不含、档期选择、取消政策 |
| **预约下单** | 选择人数（成人/儿童）、填写联系信息、优惠券/余额抵扣、微信支付 |
| **我的订单** | 订单列表（按状态筛选）、核销码展示、订单详情 |
| **个人中心** | 用户资料编辑、手机号绑定、相册管理、优惠券、邀请好友 |
| **充值中心** | 三档会员充值（银卡/金卡/钻石）、余额管理、充值记录 |
| **邀请好友** | 生成邀请码、查看邀请记录、邀请奖励 |

### 3.2 管理端（小程序内嵌）

| 功能模块 | 说明 |
|:---|:---|
| **管理面板** | 今日概览（订单数、营收、待核销人数） |
| **核销** | 手动输入核销码核销、核销统计 |
| **活动管理** | 创建/编辑活动、管理团期 |
| **团期管理** | 编辑/取消团期、自动退款、变更通知 |
| **财务统计** | 按日/周/月统计收支、按团期对账、退款订单列表 |
| **门店信息** | 编辑门店资料 |

## 四、业务流程图

### 4.1 用户预约流程

```
用户打开小程序
  │
  ├── 首页：看到门店信息 + 本周活动列表
  │
  ├── 点击某活动 → 活动详情
  │     ├── 行程安排
  │     ├── 费用包含 / 不包含
  │     ├── 选择档期（日期 + 剩余名额）
  │     └── 取消政策
  │
  ├── 点击"立即报名" → 预约下单页
  │     ├── 选择人数（成人 / 儿童）
  │     ├── 系统自动计算优惠（同行优惠 / 会员折扣 / 邀请立减 / 优惠券 / 余额抵扣）
  │     ├── 填写联系信息
  │     └── 确认支付 → 微信支付
  │
  ├── 支付成功 → 获得核销码（6 位数字 + 二维码）
  │
  └── 出行当天 → 出示核销码 → 店主扫码核销 → 完成
```

### 4.2 订单状态机

```
         支付成功              核销完成
PENDING ───────→ PAID ───────→ COMPLETED
  │                                ↑
  │ 取消/退款                      │
  ↓                                │
CANCELLED / REFUNDED          团期结束自动完成
```

### 4.3 邀请裂变流程

```
老用户 A 分享邀请码
  │
  ├── 新用户 B 通过邀请码注册
  │     ├── B 获得 ¥15 首单立减券
  │     └── 系统记录邀请关系
  │
  └── B 完成首单支付
        ├── A 获得 ¥20 余额奖励
        └── 邀请关系标记为已完成
```

## 五、产品亮点与差异化

### 5.1 场景精准

- 不是通用旅游平台，而是**以渔具店为信任单元**的垂直场景
- 目标用户明确：渔具店老客人，有天然信任基础和复购动力
- 活动内容围绕钓鱼展开，不需要复杂的品类管理

### 5.2 门槛极低

- 用户无需下载 App，微信小程序即用即走
- 店主无需额外硬件，手机即可管理活动、核销订单
- 技术上零运维，云开发自动扩缩容

### 5.3 增长闭环

- **老带新裂变**：邀请码机制，邀请人得余额奖励，被邀请人得首单立减
- **复购激励**：完成出行自动赠送复购券
- **会员体系**：充值升级，享受出行立减 + 装备折扣

### 5.4 数据可追溯

- 每笔订单有完整记录：下单时间、支付时间、核销时间
- 财务统计自动汇总，按团期对账
- 团期变更自动记录变更日志

### 6.3 与正式版（yule-go）的关系

| 维度 | cloud-mvp | yule-go |
|:---|:---|:---|
| 定位 | 快速验证 MVP | 正式产品 |
| 后端 | 云函数（Node.js） | Go (Gin) |
| 数据库 | 云数据库（文档型） | MySQL（关系型） |
| 管理后台 | 小程序内嵌 | Vue3 独立后台 |
| 扩展性 | 受限于云开发 | 自由扩展 |
| 运维 | 零运维 | 需要服务器 |

**迁移路径**：MVP 验证通过后，将云数据库数据导出，导入 MySQL，小程序前端切换 API 地址指向 Go 后端。
